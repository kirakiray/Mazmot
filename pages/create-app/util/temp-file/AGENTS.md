# 开发指南

本项目暂时是一个初始化的 ofa.js 项目，后续迭代开发时，请及时更新此处文档说明。

## 开发规则

- 本项目基于 **ofa.js** 框架开发，开发前请务必查阅 ofa.js 的 skill 或官方文档
- 如本地未安装 ofa.js skill，可从 [ofajs-docs.zip](https://raw.githubusercontent.com/ofajs/ofa.js/main/skills/ofajs-docs.zip) 下载
- 项目通常已在运行环境中，无需额外启动静态服务器

## 项目结构

```
├── /comps        # 可复用组件模块目录
├── /pages        # 页面模块目录
└── index.html    # 应用入口文件
```

### `/comps` - 组件目录

存放项目中的公共可复用组件模块：

- **组件包结构**：每个组件存放在独立目录中（组件包），组件包名通常与组件名一致
- **组件文件**：组件包内包含 `.html` 组件文件，文件名即为组件名
- **配套资源**：组件包可包含样式文件、脚本文件、使用文档和示例代码
- **组件包说明**：组件包内可包含说明文档
  - `README.md`：面向开发者的文档，包含组件功能、API说明、使用示例等
  - `SKILL.md`：面向 AI 助手的文档，包含开发规则、注意事项、最佳实践等
- **组件组合**：一个组件包可能包含多个相关组件（如 `list` 组件包可能包含 `list` 和 `list-item` 组件）
- **示例页面**：组件包内可包含演示页面，且放在独立的目录中，用于展示组件使用方法和效果

### `/pages` - 页面目录

存放应用中的页面模块：

- **目录组织**：页面模块按应用路由结构组织，可独立文件或独立目录形式存在
- **页面资源**：页面目录可包含页面代码、样式文件、脚本文件和使用文档
- **专属组件**：页面专属组件存放在页面目录下的 `/comps` 子目录中，遵循根 `/comps` 目录相同的组织规则
