# AI Agent 开发指南

## 环境准备

- 本项目基于 **ofa.js** 框架开发，操作前请先加载 ofa.js skill 以获取框架知识。
- **加载方式**：从 [ofajs-docs.zip](https://raw.githubusercontent.com/ofajs/ofa.js/main/skills/ofajs-docs.zip) 下载并解压到技能目录。
- 项目通常已在运行环境中，无需额外启动静态服务器。

## 项目结构

```
├── /comps        # 可复用组件模块目录
├── /pages        # 页面模块目录
└── index.html    # 应用入口文件
```

### `/comps` - 组件目录

存放项目中的公共可复用组件模块：

- **组件包结构**：每个组件存放在独立目录中（组件包），组件包名通常与组件名一致
- **组件文件**：组件包根目录下包含与组件包同名的 `.html` 组件文件
- **组件包说明**：组件包内必须包含以下说明文档
  - `README.md`：面向开发者的文档，包含组件功能、API说明、使用示例等
  - `SKILL.md`：面向 AI 助手的文档，包含组件的用途和使用说明
- **配套资源**：组件包可包含样式文件、脚本文件、使用文档和示例代码
- **组件组合**：一个组件包可能包含多个相关组件（如 `list` 组件包可能包含 `list.html` 和 `list-item.html` 组件）
- **示例页面**：组件包内可包含演示页面，且放在独立的目录中，用于展示组件使用方法和效果

### `/pages` - 页面目录

存放应用中的页面模块：

- **目录组织**：页面模块按应用路由结构组织，可独立文件或独立目录形式存在
- **页面资源**：页面目录可包含页面代码、样式文件、脚本文件和使用文档
- **专属组件**：页面专属组件存放在页面目录下的 `/comps` 子目录中，遵循根 `/comps` 目录相同的组织规则

## 创建新组件

当用户需要创建新组件时，按以下步骤执行：

1. 在 `/comps` 下创建与组件同名的目录
2. 在该目录下创建同名的 `.html` 组件文件
3. 创建 `README.md` 和 `SKILL.md`
4. 如需演示，创建 `/demo` 目录并放入示例页面

### 组件文件最小模板

```html
<template component>
  <style>
    :host {
      display: block;
    }
  </style>
  <div>{{text}}</div>
  <script>
    export default async ({ load }) => {
      return {
        tag: "组件标签名",
        data: {
          text: "默认值",
        },
      };
    };
  </script>
</template>
```

### SKILL.md 最小模板

````markdown
---
name: 组件名
description: 一句话描述组件用途
---

# 组件名

## 用途

简要说明组件的用途。

## 使用方法

提供引入和使用的代码示例：

```html
<l-m src="./comps/组件名/组件名.html"></l-m>
<组件名></组件名>
```

## 注意事项

- 列出使用时的关键注意点
````
