🎯 工作原理

1. 文本向量化：将文本转换为数值向量。本演示使用简单的词频向量（TF）作为示例。 在实际应用中，通常会使用预训练的语言模型（如 BERT、GPT 等）生成更高质量的向量表示。


2. 相似度计算：使用余弦相似度衡量两个向量之间的相似程度。 余弦相似度的值范围在 [-1, 1] 之间，值越接近 1 表示两个向量越相似。


3. 检索排序：根据查询向量与文档向量的相似度进行排序， 返回最相关的文档。

⚠️ 注意事项

  • 本演示使用简单的词频向量，仅用于教学目的
  • 实际应用中应使用专业的嵌入模型（如 Transformers.js）
  • 词频向量无法捕捉语义信息，只能进行关键词匹配
  • 对于大规模文档，需要使用向量数据库进行高效检索

💡 扩展建议

  • 使用 Transformers.js 在浏览器中运行预训练模型
  • 使用 LangChain.js 构建更复杂的检索应用
  • 使用 IndexedDB 存储大量文档和向量
  • 使用 Web Workers 进行后台计算，避免阻塞 UI
