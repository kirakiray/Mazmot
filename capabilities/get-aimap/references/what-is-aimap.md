# aimap 是什么？

**aimap**（AI Map）是一个由 AI 生成的项目代码索引系统，用于为项目中的代码文件生成结构化的文档索引。aimap 的主要目的是帮助 AI 助手快速定位和理解项目文件，提高 AI 辅助开发的效率。

## 核心概念

aimap 为项目中的每个代码文件创建对应的 `.md` 文档文件，这些文档文件存储在 `.aimap/` 目录下，提供代码文件的元信息描述。

### AI 生成的文件索引

aimap 文档由 AI 自动生成和维护，包含以下特点：

1. **AI 友好**：专门为 AI 助手设计，帮助 AI 快速理解项目结构
2. **自动维护**：AI 在修改或创建代码文件后，应同步更新对应的 aimap 文档
3. **上下文传递**：为下一个 AI 进程提供项目上下文，避免重复理解项目
4. **快速定位**：AI 可以通过 aimap 快速找到相关文件，无需阅读所有代码

### 文件映射规则

```
源代码文件          →  aimap 文档
index.html          →  .aimap/index.html.md
app-config.js       →  .aimap/app-config.js.md
pages/home.html     →  pages/.aimap/home.html.md
pages/layout.html   →  pages/.aimap/layout.html.md
```

## aimap 文档结构

每个 aimap 文档包含以下结构化信息：

### 1. 文件类型

描述文件的类型，例如：

- HTML 入口文件
- JavaScript 配置文件
- ofa.js 页面模块
- 组件文件

### 2. 主要功能

简要说明文件的核心功能，例如：

- 应用程序入口，加载框架和路由系统
- 应用核心配置
- 页面布局模板

### 3. 关键组件/配置

列出文件中的关键元素：

**对于 HTML 文件：**

- 使用的自定义组件（如 o-router, o-app）
- 依赖的外部资源

**对于 JavaScript 文件：**

- 导出的变量和函数
- 配置项说明

**对于页面模块：**

- 页面结构
- 模块配置（如 parent 属性）
- 样式特点

### 4. 依赖资源

列出文件依赖的外部资源：

- 核心库（如 ofa.js）
- 插件（如 router）
- 样式文件（如 Punch-UI 全局样式）

### 5. 用途

说明文件在项目中的作用和定位。

## aimap 示例

以下是一个典型的 aimap 文档示例（来自 `app-config.js`）：

```markdown
文件类型: JavaScript 配置文件
主要功能: ofa.js 应用核心配置
导出配置:

- home: 首页路由路径，指向 ./pages/home.html
- pageAnime: 页面切换动画配置 - current: 当前页面状态 (opacity: 1, 无位移) - next: 下一页面进入动画 (从右侧淡入) - previous: 上一页面返回动画 (从左侧淡入)
  用途: 定义应用路由入口和页面过渡动画效果
```

## aimap 的作用

### 1. 项目结构理解

- 快速了解项目的整体架构
- 理解各个文件的功能和职责
- 掌握文件之间的依赖关系

### 2. 代码导航

- 通过索引快速定位相关代码
- 了解组件和模块的使用方式
- 追踪依赖关系

### 3. 文档自动化

- 自动生成项目文档
- 保持文档与代码同步
- 减少手动维护文档的工作量

### 4. AI 辅助开发

- 为 AI 工具提供项目上下文
- 帮助 AI 理解项目结构
- 提高代码生成和修改的准确性

### 创建 aimap 文档

在创建新文件时，可以同时生成对应的 aimap 文档：

```javascript
// 创建代码文件
await writeFile("pages/home.html", code);

// 创建对应的 aimap 文档
await writeFile("pages/.aimap/home.html.md", aimapContent);
```

### AI 使用 cap-request 更新文件

AI 助手可以使用 `cap-request` 创建代码文件和对应的 aimap 文档：

**步骤 1：创建代码文件**

<cap-request>
  <template
    name="fs"
    cid="fs-write-code-01"
    desc="写入代码文件"
    data-mode="write"
    data-path="pages/home.html"
  >
<!-- 代码内容 -->
  </template>
  <template
    name="fs"
    cid="fs-write-aimap-01"
    desc="写入 aimap 文档"
    data-mode="write"
    data-path="pages/.aimap/home.html.md"
  >
文件类型: ofa.js 页面模块
主要功能: 应用首页，展示欢迎信息和入门指南
页面结构:
  - 欢迎区域: 居中显示标题和简介
  - 卡片区域: 展示快速入门指南
模块配置:
  - parent: ./layout.html (继承布局模板)
用途: 作为应用默认首页，引导用户了解项目结构
  </template>
</cap-request>

## 最佳实践

### AI 开发流程中的 aimap 维护

**重要提示**：当 AI 助手修改或创建代码文件后，**必须**同步更新对应的 aimap 文档。这是确保下一个 AI 进程能够准确理解项目的关键步骤。

```
AI 开发流程：
1. AI 读取 aimap 索引，快速理解项目结构
2. AI 定位并修改相关代码文件
3. AI 同步更新对应的 aimap 文档 ✅
4. 下一个 AI 进程可以基于最新的 aimap 继续开发
```

### 其他最佳实践

1. **保持同步**：每次修改代码文件时，更新对应的 aimap 文档
2. **结构化描述**：使用统一的格式描述文件信息
3. **简洁明了**：aimap 文档应简洁但信息完整
4. **关键信息优先**：优先描述最重要的功能和组件
5. **包含依赖**：明确列出文件依赖的外部资源

## aimap 与传统文档的区别

| 特性   | aimap              | 传统文档             |
| ------ | ------------------ | -------------------- |
| 位置   | 与代码文件同目录   | 独立的文档目录       |
| 粒度   | 每个文件一个文档   | 通常按模块或功能组织 |
| 同步性 | 易于与代码保持同步 | 容易与代码脱节       |
| 用途   | 代码索引和快速理解 | 详细的功能说明和教程 |
| 自动化 | 易于自动生成和维护 | 通常需要手动编写     |

## 总结

aimap 是一个轻量级的代码索引系统，通过为每个代码文件生成结构化的元信息文档，帮助开发者和 AI 工具快速理解项目结构、定位代码位置、理解文件功能。它是项目文档体系的重要组成部分，与传统的详细文档互为补充。
