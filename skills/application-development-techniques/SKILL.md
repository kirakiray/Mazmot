# 开发应用的技巧

- 一般情况下，项目是基于 ofa.js 框架开发的，所以记得查阅一下 **ofa.js** 框架的技能文档
- 首先通过 **get-aimap** 的能力快速了解项目的结构
    - aimap 是存在于项目的文件，用于记录项目中每个文件的概览和功能说明，帮助ai快速了解项目结构
- 得到项目结构后，可以根据需求，使用 **fs** 能力读取项目中的具体文件内容
- 分析需求规划应用的需求规划应用的模块和页面
    - 将需求拆分为多个组件或页面模块，或js文件
    - 然后逐个实现每个模块或页面
    - 每个独立的组件或页面模块，下面可以建立测试用目录，独立去试用这些组件或页面模块的功能是否符合预期
    - 最后所有预测好的组件或页面模块都开发好后，将这些组件和页面模块整合到应用中，进行测试

## 调试技巧
- 因为是在模拟器中，可以使用 **emulator-console** 的能力查看应用的控制台日志的，所以写入文件的时候，适当给组件或页面模块的生命周期带上日志信息，或者事件处理函数里带上打印日志，方便AI兽面收集信息进行调试
- 写入文件后，使用 **emulator-navigate** 能力刷新一下，就可以开始调试
- 使用**emulator-inspect** 获取应用当前页面的元素状态，方便调试
- 使用**emulator-interact** 与应用当前页面的元素进行交互，方便调试
- 当你交互完成后，通过 **emulator-console** 能力查看应用的控制台日志，查看是否符合预期，是否有异常情况发生
- 有异常的状态的话，进行修复问题，写入文件，刷新，交互等重做流程，直到符合预期的结果

## UI组件开发技巧

有一些例如按钮啊，输入框什么的UI组件，可以直接使用 punch-ui 组件库，不需要自己实现。你可以查看一下 punch-ui 的 技能，看能否符合需求。
如果没有满足需求的组件，则自己进行开发，
开发的时候参考上面的调试技巧。
