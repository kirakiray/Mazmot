# 理解项目

在应用开发模式下，快速理解项目结构是高效开发的关键。本技能提供两种方法帮助你获取项目信息。

## 方法一：通过 aimap 索引（推荐）

大多数项目都包含 aimap 索引信息，这些索引记录了项目中每个文件的概览和功能说明。

**适用场景**：项目包含 `.aimap` 目录和索引文件

**操作步骤**：

调用 `emulator-aimap` 能力获取项目的所有 aimap 索引信息：

<cap-request>
  <template name="emulator-aimap" cid="understand001" desc="获取项目的 aimap 索引信息" data-project-path="your-project-path">
  </template>
</cap-request>

返回的数据将包含项目中所有文件的路径、功能描述和详细说明，帮助你快速了解项目结构。

## 方法二：直接读取文件

如果项目没有 aimap 索引信息，可以使用 `fs` 能力直接读取项目文件。

**适用场景**：项目缺少 aimap 索引或索引不完整

**操作步骤**：

1. 使用 `fs` 能力列出项目目录结构
2. 读取关键文件（如配置文件、入口文件等）
3. 分析文件内容，理解项目架构

这种方法虽然更耗时，但可以获取最完整的项目信息。

## 最佳实践

- **优先使用 aimap 索引**：快速、结构化、包含功能说明
- **结合两种方法**：先查看 aimap 了解整体结构，再读取关键文件获取细节
- **关注核心文件**：配置文件、入口文件、路由文件等通常是理解项目的关键
