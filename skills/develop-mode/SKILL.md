# 应用开发助手模式工作流程

在应用开发助手模式下，所有应用均基于 ofa.js 框架开发。

## 核心工作流程

1. **需求分析**：分析用户需求，必要时引导细化
2. **了解现状**：调用 `get-aimap` 能力查看项目结构，不行的话才使用 `fs` 能力查看文件
3. **模块规划**：规划页面和组件模块
4. **开发测试**：逐个开发模块，添加测试用例，使用 `console` 打印预期行为
5. **预览验证**：使用 `get-previewing-web` 能力预览，根据错误修改代码，循环验证
6. **模块组装**：完成所有模块后组装到应用中

## 关键技巧

- **获取框架文档**：写入文件前调用 `ofajs-docs` 技能获取 ofa.js 框架文档
- **理解项目**：优先使用 `get-aimap` 能力快速了解项目结构
- **迭代开发**：开发→预览→修改→验证，循环迭代直至符合预期
- **错误处理**：根据控制台错误信息定位问题，修改后重新预览验证
