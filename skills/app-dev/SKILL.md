# 应用开发技巧

## 项目认知

- 项目通常基于 **ofa.js** 框架开发，开发前应查阅 ofa.js 技能文档
- 使用 **get-aimap** 能力快速了解项目结构（aimap 记录了项目中每个文件的概览与功能说明）
- 了解结构后，使用 **fs** 能力读取具体文件内容

## 开发流程

1. **需求分析** — 将需求拆分为独立的组件、页面模块或 JS 文件
2. **逐个实现** — 依次开发每个模块，每个独立模块下可建立测试目录，单独验证功能
3. **集成测试** — 所有模块开发完成后，整合到应用中进行整体测试

## 调试流程

应用运行在模拟器中，可通过以下能力形成完整的调试闭环：

1. **写入代码** — 在组件生命周期和事件处理函数中添加日志，便于排查问题
2. **刷新页面** — 使用 **emulator-navigate** 刷新应用，使改动生效
3. **检查状态** — 使用 **emulator-inspect** 获取当前页面的元素状态
4. **模拟交互** — 使用 **emulator-interact** 与页面元素进行交互
5. **查看日志** — 使用 **emulator-console** 查看控制台输出，确认是否符合预期
6. **修复迭代** — 若有异常，修复代码后回到步骤 1，直到结果符合预期

## UI 组件开发

- 优先使用 **punch-ui** 组件库（按钮、输入框等基础组件无需自行实现），查阅 punch-ui 技能文档确认是否满足需求
- 若组件库无法满足，则自行开发，开发时遵循上述调试流程
